## 專案名稱
空氣品質指標資料分析系統

## 專案簡介
本專案透過空氣品質指標（AQI）資料集，實現了動態顯示空氣品質數據的網頁系統，讓用戶能夠篩選特定縣市的空氣品質資料並檢視相關圖表。

## 資料集來源
使用的資料集為台灣環境部提供的空氣品質指標資料集：
[空氣品質指標資料集 API](https://data.moenv.gov.tw/api/v2/aqx_p_432?api_key=e8dd42e6-9b8b-43f8-991e-b3dee723a52d&limit=1000&sort=ImportDate%20desc&format=JSON)

## 系統架構
- 後端：使用 AWS 提供的 MySQL (RDS) 儲存空氣品質資料。
- 前端：建立網頁介面，讓用戶可互動地篩選數據及檢視圖表。

## 功能特性
1. 資料庫設計：
   - 在 AWS 創建 MySQL 資料庫，用於儲存從 JSON API 匯入的資料。
   - 使用 SQL 指令檢視資料表內容。
2. 前端功能：
   - 動態篩選縣市空氣品質資料。
   - 顯示對應縣市的空氣品質圖表。
3. 後端處理：
   - 使用 Node.js 連接資料庫，實現數據讀取和後端邏輯。

## 使用方法
1. **設置資料庫**：
   - 登入 AWS RDS，創建免費方案的 MySQL 資料庫。
   - 匯入空氣品質 JSON 資料到 MySQL。
   - 使用 SQL 指令檢視資料表：`SELECT * FROM air_quality;`
2. **啟動應用**：
   - 在終端中執行指令：
     ```
     mysql -h air-quality-sql.cb6066o8cpi6.ap-northeast-3.rds.amazonaws.com -P 3306 -u admin -p
     ```
   - 登入資料庫後，選擇資料庫並檢視資料：
     ```
     USE air_quality_data;
     ```
   - 啟動後端程式：
     ```
     node {檔案名稱}
     ```
3. **使用網頁**：
   - 成功啟動後，訪問提供的網址。
   - 使用者可透過篩選功能選擇特定縣市，並檢視對應的數據與圖表。

## 系統功能展示
- 動態圖表範例：
  以台北市為例，用戶能即時檢視空氣品質資料的動態變化及對應數據。
